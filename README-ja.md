Claude Code などのインタラクティブな AI CLI を包む、**完全透過の PTY プロキシ**。AI が静かになった瞬間に知らせてくれる。

プロンプトを送ったら、その場を離れていい。`crai` は静かに見張り、AI が終わったときだけ口を開く。

---

## インストール

```sh
brew install your-name/tap/crai
```

ソースからビルドする場合：

```sh
git clone https://github.com/your-name/crai
cd crai
go build -o crai .
sudo mv crai /usr/local/bin/
```

---

## 使い方

```sh
# claude を直接ラップする
crai claude

# 引数はそのまま透過される
crai claude --dangerously-skip-permissions
```

すべてそのまま透過される。色、スピナー、キーバインド——すべて無傷。`crai` は必要な時まで見えない。

---

## エイリアス設定

シェルの設定ファイル（`~/.zshrc` または `~/.bashrc`）に追加：

```sh
alias claude="crai claude "
```

> **末尾のスペースについて**
> bash と zsh では、エイリアスの値の末尾にスペースがあると、次の単語もエイリアスとして展開される。これにより、`claude` の後に渡す引数もエイリアス展開の対象になり、エイリアスチェーンが完全に機能する。

あとはいつも通り `claude` を使うだけ。`crai` は静かに見張っている。

## 仕組み

```
 ┌──────────┐   raw stdin   ┌─────────────┐   PTY   ┌───────────┐
 │  あなた  │ ────────────► │    crai     │ ──────► │  claude   │
 │  (人間)  │ ◄──────────── │  (watcher)  │ ◄────── │  (AI CLI) │
 └──────────┘   raw stdout  └──────┬──────┘         └───────────┘
                                   │
                   AI 出力から 1500ms 以上の沈黙
                                   │
                                   ▼
                    🔔 afplay Glass.aiff
                    🪟 通知センターバナー
                    🔕 ターミナルベル (\a)
```

1. コマンドを**疑似端末（PTY）**の中で起動する
2. raw な stdin/stdout を変換なしでブリッジする
3. 出力ストリームの**沈黙**を監視——1500ms 以上新しい出力がなければ AI が完了したと判断
4. 完了時：サウンド、通知センターバナー、ターミナルベルの3つを同時に発火

### スマートフィルタリング

- **1対1プロンプト対応** — Enter を押すたびに通知が1つだけ予約される。対応するプロンプトのない AI 出力（起動時のバナー、自発的な出力）は無視される
- **エコー除去** — キー入力から 100ms 以内に届いた出力は PTY のエコーとして扱い、AI 出力とみなさない
- **短時間応答の抑制** — AI が 5 秒以内に応答した場合は通知しない（まだ見ているはず）
- **入力中の抑制** — 次のメッセージを作成中は通知しない

## オプション

| フラグ | 説明 |
|--------|------|
| `--no-banner` | 通知センターバナーを無効化 |
| `--no-sound` | サウンドを無効化 |

## 動作環境

- macOS（音声に `afplay`、通知センターに `osascript` を使用）
- 任意のコマンドライン AI ツール（または他の長時間実行インタラクティブ CLI）
